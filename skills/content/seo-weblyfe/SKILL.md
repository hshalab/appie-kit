# Weblyfe SEO

Umbrella skill for the Weblyfe SEO curriculum. Prefer this over one-module-per-skill micro entries.

## Canonical workflow
1. Start with the ranking model and site goal.
2. Choose target keywords and map one primary keyword per URL.
3. Apply the on-page checklist.
4. Confirm technical crawl/index health.
5. Build internal-link architecture and content clusters.
6. Grow authority with relevant backlinks.
7. Re-audit based on Search Console, rankings, and CTR changes.

## Module map
### Foundation
- Why SEO matters
- The six Google signals: relevance, popularity, uniqueness, freshness, quality, readability
- SEO as both traffic acquisition and reputation management

### Keyword strategy
- Long-tail preference
- Search-volume validation before writing
- One primary keyword per page
- Early placement in title, H1, intro, headings, URL, meta, and alt text

### On-page checklist
- Title tag, meta description, slug, H1, first paragraph, heading placement
- Image filenames and alt tags
- Breadcrumbs and media enrichment

### Technical SEO
- Sitemap submission
- Search Console coverage monitoring
- mobile-first rendering
- speed, schema/JSON-LD, canonicals, plugin/app hygiene

### Smart content architecture
- Pillar + supporting article clusters
- Internal links as a deliberate authority network
- Retargeting and conversion instrumentation on content assets
- Favor 2 posts per week when the cluster map is active, rather than publishing isolated one-offs
- Use the blog as a support system for conversion pages, not just a publishing feed
- Build posts from market research plus internal learnings, not from a pre-made calendar.
- Match Seyed's voice: direct, authentic, no corporate fluff.
- Avoid em dashes.
- Use good sources and backlinks.
- Current Weblyfe blog operating mode: 2 posts per week, written from market research plus internal learnings, with a decision-stage bias when the cluster supports it.
- Live publication standard: every published Weblyfe blog must include a branded cover image and set both CMS image fields (`main-image` and `thumbnail-image`). Use `references/weblyfe-blog-publish-and-cover-checklist.md`.
- Blog drafts are not "done" until the article is placed in the Weblyfe CMS, not just saved as markdown.
- When the user explicitly asks to push live, publish in Webflow CMS immediately, then verify the public URL (HTTP 200 + correct title/snippet) before confirming.
- When repurposing the same idea for LinkedIn, Dutch is acceptable if requested.
- See `references/weblyfe-editorial-system-2026.md` for the current cluster-based publishing rhythm and topic directions.
- See `references/weblyfe-content-consistency-operating-system-2026.md` for the weekly operating cadence that connects 2 blogs/week, LinkedIn drafts, repurposing, and review loops.
- See `references/weblyfe-webflow-blog-publish-runbook.md` for the Webflow CMS go-live sequence and verification checklist.
- See `references/weblyfe-editorial-voice-and-cadence-2026.md` for the current voice, cadence, and LinkedIn language preference.

## Content audit workflow
1. Inventory the live archive and group posts into clusters.
2. Identify the strongest commercial or high-intent post in each cluster.
3. Check whether each post reinforces a target page or conversion path.
4. Mark thin, standalone, or overly broad posts for rewrite or cluster support.
5. Re-run the audit after new content lands or rankings shift.
- See `references/weblyfe-blog-audit-2026.md` for the latest live archive analysis and cluster map.
- See `references/surferseo-webflow-copy-audit.md` for auditing Webflow pages against SurferSEO Content Editor drafts, including the GraphQL permalink-hash extraction pattern and report shape.

### Backlinks
- Authority + topical relevance over raw quantity
- Guest posts, owned properties, social profiles, linkable assets
- Avoid link farms, bought links, and scaled reciprocal-link schemes
