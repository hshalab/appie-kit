# Review gate notes

Use this as the concrete pack format when the user wants approval before continuing with creative production.

## Pack contents
- Title and goal
- Short script or narration draft
- Frame/beat list with IDs and one-line purpose
- Aspect ratio or format callout
- Locked visual canon or brand constraints
- CTA or end-card direction
- Explicit question asking what to change, keep, or approve

## Verification mindset
- Preserve approved anchors.
- Only adjust the smallest adjacent pieces when the user requests changes.
- Do not continue into expensive downstream work until approval is clear.

## Common failure modes
- Continuing silently after the user asked for curation
- Regenerating the whole concept when only one frame or line changed
- Mixing draft, approved, and pending assets without labels
