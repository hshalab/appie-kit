# Live Source Hierarchy for Reels-Only Opportunity Scans — 2026-06-27

This is the verified source order from the June 27 scan run.

## Preferred source order
1. Search Engine Journal RSS for AI search, SEO, agent visibility, and search platform shifts.
2. n8n RSS for agent reliability, routing, governance, memory, and production automation patterns.
3. Webflow Updates pages for platform/product shifts that can be turned into agency positioning.
4. Public competitor homepages, pricing pages, and blog indexes when a named comparator is needed.

## Minimum evidence per signal
- Title
- Date or recentness cue
- One-line angle summary
- Canonical URL

## Extraction notes
- If the browser shell is skeletal or slow, switch to terminal HTTP/requests after a couple of failed attempts.
- For Webflow update detail pages, trust the visible Launched on label when meta/canonical fields are missing.
- For Webflow listing pages, the HTML is enough to recover recent update titles and slugs.
- For RSS feeds, use feed items directly before fetching article pages.

## Ranking rule for reels-only scans
Rank by:
1. watch-time potential
2. shareability
3. proof density
4. why-now freshness

Keep stale local context separate and label it by date in the final write-up.
