# Short-Form Content Intelligence

Use this skill when the task is to research, rank, or convert short-form video opportunities for Weblyfe/Appie or WebU.

## Core job
Turn live signals and local context into reel-ready opportunities, ranked by:
1. watch-time potential
2. shareability
3. proof density
4. why-now freshness

## When to use
- Daily opportunity scans
- Competitor hook research
- Scan-to-brief conversions
- Reels-only weekly calendars
- Live trend sweeps for Appie/Weblyfe or WebU

## Hard rules
- Reels only. Do not suggest carousels, single-image posts, static graphics, or mixed-format calendars.
- Keep the output decision-ready, not strategic fluff.
- Preserve the live ranking unless a later item is clearly stronger on watch-time, shareability, proof density, or why-now freshness.
- Separate live signals from stale local context and label stale context by date.
- For each opportunity, keep the evidence tiny: title, date or recency cue, one-line angle, canonical URL.

## Two-stream structure
When both audiences are in scope:

### Stream A - Appie/Weblyfe-corp
Use for @appie.ai and @weblyfe.
Focus on:
- AI automation
- Webflow/platform shifts
- SEO/AEO visibility
- reliability, observability, routing, grounding
- proof-led authority content

### Stream B - WebU-tinted
Use for @seyed.jpg and @weblyfeuniversity.
Focus on:
- 19-24 year-old creatives
- no-code agency path
- accountability, overwhelm, imposter syndrome
- client acquisition and premium positioning
- visible systems and student outcomes

For Stream B, keep the 3 pillars explicit:
- Authoritative
- Educational
- Value-Driven

Recommended distribution for a 5-reel batch:
- 2 Educational
- 2 Authoritative
- 1 Value-Driven

## Live source order
Prefer this source order for fresh signals:
1. Search Engine Journal RSS for AI search, SEO, and visibility shifts
2. n8n RSS for agent reliability, routing, governance, and automation patterns
3. Webflow Updates for product/platform shifts that can be reframed into agency positioning
4. Public competitor homepages, pricing pages, and blog indexes when named comparators are needed

If the browser shell is skeletal or slow, switch quickly to terminal HTTP / requests rather than looping browser retries.

## Minimum evidence standard
For each live signal, capture:
- title
- date or recentness cue
- one-line summary or angle
- canonical URL

For Webflow detail pages, trust the visible Launched on label when metadata is missing.

## Scan-to-brief conversion
When turning a scan into a production brief:
- keep the original ranking intact
- make the reel shot-ready
- include target account, audience pain, hook, payoff, why now, proof/source, execution concept, and score
- end with the top 1-2 ideas worth producing today

## Verification mindset
When live and local sources disagree, trust the newer live signal for ranking and use local context only to sharpen framing.
