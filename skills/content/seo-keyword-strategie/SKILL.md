# SEO Keyword Strategie Skill

## Purpose
Select, validate, and place the right keywords for any Weblyfe site or blog post. Use this before writing any content, when auditing an existing page that is not ranking, or when planning a new site's content architecture. Codifies the long-tail focus and keyword placement rules from Seyed's course.

## Source
"Scoren op zoekwoorden" PDF from Seyed's Weblyfe University SEO course (downloaded from Drive 2026-05-03).

## Quick reference
| Task | Action |
|------|--------|
| Find keyword ideas | Ubersuggest (ubersuggest.com) - primary tool named in PDF |
| Monitor current rankings | Google Search Console > Performance > Queries |
| Track competitor keywords | SEMrush |
| Find long-tail suggestions | Google's "Zoekopdrachten gerelateerd aan..." at bottom of SERP |
| Place keyword in content | H1, first paragraph, H2/H3, URL, title tag, meta description, alt tags |

## Core principles
1. "Hoe verder naar voren een woord staat, hoe meer waarde het van Google krijgt." - keyword position in a sentence matters.
2. Long-tail keywords (specific phrases) make up 70% of all searches. Short head terms make up only 30% and carry far more competition.
3. Avoid keyword stuffing. The example in the PDF ("SEO blog SEO tips SEO trucs SEO...") is explicitly mocked as spam that Google penalises.
4. One primary keyword per page. Supporting keywords can appear but one term should dominate each URL.
5. Moving from position 2 to 1 = +28.4% CTR gain. Every position matters; target the top 5.
6. Monsterartikelen (monster articles) are the highest-value content format: long, with images, video, outgoing links, interactie, backlinks, and internal links combined.
7. Use tools to validate search volume before writing. Intuition alone is not enough.
8. "Lekker specifiek" - the more specific the query, the less competition and the higher the buyer intent.

## How to apply (for any new Weblyfe site)
- Open Ubersuggest, enter the main topic of the page/site, collect 10-20 candidate keywords.
- Filter for long-tail variants (3+ words). Favour specificity over volume.
- Check Google's "gerelateerde zoekopdrachten" at the bottom of the SERP to find buyer-intent variants.
- Assign one primary keyword per page/post. Map keywords to URLs in a spreadsheet before writing.
- Place the primary keyword: title tag, H1, first 100 words of body, at least one H2 or H3, URL slug, meta description, featured image filename and alt tag.
- Validate with Google Search Console after 4-8 weeks: check impressions and average position for the target query.
- Use SEMrush to identify which keywords competitors rank for that you do not yet cover.

## Anti-patterns (what the PDF says NOT to do)
- Keyword stuffing: repeating the keyword unnaturally throughout the text. Google treats this as spam.
- Targeting only head terms (single words like "aandelen"). Competition is extreme and intent is unclear.
- Writing content without checking search volume first. Effort with no audience = zero ROI.
- Ignoring related searches. The long-tail goldmine is in the "Zoekopdrachten gerelateerd aan" section Google shows.

## Examples and templates
Long-tail example from PDF:
- Head term: "Aandelen" (hoge concurrentie, 30% of queries)
- Mid: "Kooptips aandelen"
- Long-tail: "Welke goedkope aandelen kan ik nu het beste kopen?" (lage concurrentie, part of the 70%)

Monsterartikel checklist (from PDF slide):
- Afbeeldingen (images with alt text)
- Video (embedded or linked)
- Lengte (long-form)
- Uitgaande links (outbound to authority sources)
- Interactie (comments, shares, time-on-page signals)
- Backlinks (earned inbound)
- Interne links (internal linking network)
